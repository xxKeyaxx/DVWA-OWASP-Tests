Test Overview

OWASP Testing Guide - OTG-CONFIG-006

Objective: Test HTTP Methods

Description: Verify that the web server and application properly handle HTTP methods and do not expose dangerous methods that could be exploited.

Target: http://localhost/dvwa

Methods Tested: 16 HTTP methods

Endpoints Tested: 18 DVWA endpoints

Total Tests Performed: 294

Methodology

The testing methodology included the following steps:

  1. Authentication: Logged into DVWA with default credentials
  2. Method Enumeration: Tested 16 HTTP methods including:
    • Standard methods: GET, POST, HEAD, OPTIONS
    • Extended methods: PUT, DELETE, TRACE, PATCH, CONNECT
    • WebDAV methods: PROPFIND, PROPPATCH, MKCOL, COPY, MOVE, LOCK, UNLOCK
  3. Endpoint Testing: Tested methods on 18 key DVWA endpoints including:
    • Main application pages
    • Authentication interfaces
    • Security configuration pages
    • Vulnerability modules
  4. Method Tampering: Tested HTTP method bypass techniques
  5. Risk Assessment: Analyzed responses for security implications

HTTP Methods Overview

Standard HTTP Methods

  • GET - Retrieve information (Generally safe)
  • POST - Submit data (Generally safe)
  • HEAD - Retrieve headers only (Generally safe)
  • OPTIONS - Query available methods (Informational)

Dangerous HTTP Methods

  • TRACE - Echoes back the received request (High risk - XST vulnerability)
  • PUT - Upload file/resource (High risk)
  • DELETE - Delete resource (High risk)
  • CONNECT - Reserved for proxies (High risk)
  • PATCH - Partial resource modification (Medium risk)

WebDAV Methods

  • PROPFIND - Retrieve properties (Medium risk)
  • PROPPATCH - Modify properties (Medium risk)
  • MKCOL - Create collection/directory (Medium risk)
  • COPY/MOVE - File operations (Medium risk)
  • LOCK/UNLOCK - Resource locking (Medium risk)

Remediation Recommendations

  1. Disable Dangerous Methods: Explicitly disable HTTP methods that are not required for application functionality
  2. Web Server Configuration: Configure web server to reject dangerous HTTP methods
  3. Application-Level Controls: Implement proper method validation in the application
  4. Regular Auditing: Periodically test HTTP methods to ensure they remain properly restricted
  5. Security Headers: Implement proper security headers to prevent method abuse

Apache Configuration Example:

# Disable dangerous HTTP methods

Require all denied


# Explicitly deny TRACE method
TraceEnable Off

Nginx Configuration Example:

# Block dangerous HTTP methods
if ($request_method !~ ^(GET|HEAD|POST|OPTIONS)$ ) {{
return 405;
}}

# Disable TRACE method
more_clear_headers ' TRACE';
